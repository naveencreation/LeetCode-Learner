Binary Tree · Traversal

Inorder Traversal

Visit nodes in Left → Root → Right order. For a BST, this gives you a sorted sequence.

Recursion DFS Interview Essential

Key concepts at a glance — for those who already know the basics.

01 · The rule

Left → Root → Right

For BSTs, this always produces values in ascending sorted order.

02 · How to think

1
Keep going left until you hit None
Dive to the deepest left leaf first.
2
Visit (record) the current node
Only after the full left subtree is done.
3
Repeat the same logic for the right subtree
Apply Left → Root → Right recursively.

03 · Code

def inorder(node, result):
    if node is None: return # base case
    inorder(node.left, result)  # 1. left
    result.append(node.val)     # 2. root
    inorder(node.right, result)# 3. right

04 · Complexity

Time
O(n)
Every node visited once.
Space
O(h)
Stack depth = tree height.

05 · Interview uses

Kth smallest in BST — kth element in inorder sequence.

Validate a BST — inorder must be strictly increasing.

Recover BST — find where inorder order breaks.

01 · Start here — what even is a binary tree?

Build the foundation first

Before traversal makes sense, you need to picture the structure. A binary tree is a collection of nodes where each node has at most two children — called the left child and the right child. The topmost node is called the root. Nodes with no children are called leaves.

The blue node is the root. The green nodes are leaves — they have no children. Every other node is an internal node.

02 · What does "traversal" mean?

Visiting every node — in what order?

Traversal simply means visiting every node in the tree exactly once. The question is — in what order do you visit them? There are three classic DFS orders:

Preorder

Root→L→R

Inorder ← you are here

L→Root→R

Postorder

L→R→Root

Inorder means: for any node, always finish the entire left subtree first, then visit that node, then finish the entire right subtree.

03 · The real-world analogy

Think of it like reading a book shelf

Imagine a bookshelf where books are arranged from left to right in sorted order. Inorder traversal is exactly like reading them in that order — you always finish everything to the left before reading the current book, and only then move to the right. The "current book" (root) sits between everything on its left and everything on its right.

The BST superpower: If the tree is a Binary Search Tree (BST), inorder traversal always gives you values in ascending sorted order. This is one of the most tested facts in tree interviews.

04 · Understanding recursion here

What is the function actually doing?

Recursion is when a function calls itself. The trick is that each call handles one node, and trusts that the same function will correctly handle the subtrees. Here is how to read the code line by line:

def inorder(node, result):
    if node is None: return
    inorder(node.left, result)
    result.append(node.val)
    inorder(node.right, result)
Why do we check if node is None: return?
This is the base case — it stops the recursion. Without it, the function would keep calling itself forever. When a node has no left child, node.left is None, so we call inorder(None) which immediately returns. This is how recursion knows when to stop going deeper.
Why does the append line come in the middle — not first or last?
Because inorder means Left first, then Root, then Right. Appending in the middle means we only record the current node after the entire left subtree is finished. If you moved the append line to the top, you'd get preorder. If you moved it to the bottom, you'd get postorder.
What does the recursion stack look like at the deepest point?
When we first reach node 4 (the leftmost leaf), the call stack looks like this — inorder(4) is on top, waiting to run:
→ inorder(node=4)  ← currently here
   inorder(node=2)
   inorder(node=1)  ← original call

05 · Full trace — every step explained

Walking through the example tree

Let's trace every single action the algorithm takes on this tree, narrated step by step.

Going

Enter inorder(1) — the root

We start at 1. Rule says go left first, so we call inorder(1.left) = inorder(2). Node 1 is now waiting.

Going

Enter inorder(2)

At node 2. Go left first → call inorder(2.left) = inorder(4). Node 2 is now waiting.

Going

Enter inorder(4) — leftmost leaf

At node 4. Try to go left → inorder(None) → base case, returns immediately. Now visit node 4.

Visit

Record 4 → result = [4]

Node 4 has no left, no right. We're done with it. Return back to node 2.

Visit

Record 2 → result = [4, 2]

Left subtree of 2 is finished. Now visit node 2 itself. Then go right → inorder(5).

Visit

Record 5 → result = [4, 2, 5]

Node 5 has no children. Visit it and return. Left subtree of root (1) is now fully done.

Visit

Record 1 → result = [4, 2, 5, 1]

Entire left subtree finished. Now visit the root itself. Then go right → inorder(3).

Visit

Record 6 → result = [4, 2, 5, 1, 6]

Enter node 3, go left to 6. Node 6 is a leaf. Visit it and return to 3.

Visit

Record 3 → result = [4, 2, 5, 1, 6, 3]

Left of 3 is done. Visit node 3. Go right → inorder(7).

Visit

Record 7 → result = [4, 2, 5, 1, 6, 3, 7] ✓

Node 7 is a leaf. Visit it. All done — this is the final inorder sequence.

06 · Common beginner mistakes

What trips people up

✕

Forgetting the base case

Without if node is None: return, the function crashes with a recursion error when it tries to access None.left.

✕

Mixing up the order

Moving the append before the left call gives preorder. Moving it after both calls gives postorder. Position of append defines the traversal.

✕

Thinking inorder = sorted for all trees

Inorder only gives sorted output for a BST. For a random binary tree, the output order depends entirely on node placement.

✕

Not passing result by reference

In Python, lists are passed by reference — all recursive calls share the same list. If you create a new list inside each call instead, results will be lost.

07 · Complexity — the full explanation

Why O(n) time and O(h) space?

Time
O(n)
We visit every node exactly once — no node is skipped or revisited — so time grows linearly with the number of nodes.
Space
O(h)
The call stack holds one frame per level of depth. h = log n for balanced trees, h = n for skewed (like a linked list).

Interview tip: If they ask "what is the space complexity?", always clarify — O(log n) for a balanced tree, O(n) worst case for a skewed tree. Showing you know both impresses interviewers.

08 · Interview context

Where you'll actually use this

Kth smallest element in BST

Inorder gives sorted order. Count to the kth element — done. No sorting needed.

Validate a Binary Search Tree

Run inorder. If the result is not strictly increasing at every step, the BST property is violated.

Convert BST to sorted array

Simply collect the inorder result. That is your sorted array — O(n) time, no extra sorting.

Recover BST (two nodes swapped)

Run inorder. Find where the sequence breaks (a larger value before a smaller one). Those two nodes are the swapped pair.
