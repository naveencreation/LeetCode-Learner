Symmetric Tree

Check whether a binary tree is a mirror of itself — the left subtree mirrors the right subtree at every level.

Recursive DFS Mirror Check BFS Variant Easy

Key concepts at a glance — for those who already know the basics.

01 · The rule

A tree is symmetric if its left and right subtrees are mirrors of each other

Two subtrees are mirrors when: their roots have equal values, the left child of one matches the right child of the other, and vice versa — checked recursively all the way down.

02 · How to think

1
Start with the root's two children
Call a helper isMirror(left, right) with root.left and root.right.
2
Check the base cases first
Both None → symmetric ✓. One None, one not → asymmetric ✗. Both exist → compare values.
3
Recurse with the outer and inner pairs
Outer: left.left vs right.right. Inner: left.right vs right.left. Both must pass.
4
Return True only when values match AND both recursive calls pass
Any mismatch anywhere short-circuits the whole check to False.

03 · Code (Python)

def isSymmetric(root) -> bool:
 
def isMirror(left, right) -> bool:
if not left and not right: # both None → symmetric
return True
if not left or not right: # one None → not symmetric
return False
 
return (
left.val == right.val # values match
and isMirror(left.left, right.right) # outer pair
and isMirror(left.right, right.left) # inner pair
)
 
return isMirror(root.left, root.right)

Same values — different structure. The extra child appears on the same side both times, breaking symmetry.

01 · What does "symmetric" mean for a tree?

The concept from scratch

A binary tree is symmetric if it looks the same when you fold it along a vertical axis through the root. This means the left subtree of the root is a mirror image of the right subtree — not just the values, but the entire shape and structure must be mirrored too.

Key insight: symmetry is not about any single node's children — it's a structural relationship between two separate subtrees. You always compare two nodes at a time: one from the left branch and its corresponding mirror node from the right branch.

Two subtrees are mirrors of each other when: their root values are equal, the left child of the left subtree mirrors the right child of the right subtree (outer pair), and the right child of the left subtree mirrors the left child of the right subtree (inner pair). This definition is naturally recursive.

02 · Outer pair vs Inner pair

Understanding which children to compare

This is the most confusing part for beginners. When you compare two mirror nodes L and R, you don't compare L's left with R's left — you cross the axis. Here's the rule visualised:

Memory trick: Outer pair = both point away from the axis (far-left vs far-right). Inner pair = both point toward the axis (near-right vs near-left). Always cross the axis — never compare same-side children.

03 · Why recursive DFS?

The traversal that fits this problem naturally

Symmetry is a property that must hold at every node simultaneously. The most natural way to verify this is to recurse: check the current pair's values, then verify that both child-pairs are also mirrors. The answer at any node depends on results from its children — which is exactly what recursion provides.

Recursive DFS ← natural fit

Each call handles one mirror pair. Return values bubble results back. Short-circuit and stops early on first failure. Minimal code.

BFS / Iterative

Queue of pairs instead of call stack. Same logic, no recursion limit risk. Useful for very deep trees in Python (default limit ~1000 frames).

Interview tip: Lead with the recursive solution — it's cleaner and easier to explain. Then mention the iterative BFS alternative and note it avoids Python's recursion limit. Both are O(n) time and O(h) / O(w) space respectively.

04 · BFS / Iterative alternative

Solving it without recursion

Instead of using the call stack, you explicitly manage a queue of pairs to check. Each iteration pops one pair, applies the same three conditions, then pushes the outer and inner child pairs for later checking.

from collections import deque
 
def isSymmetric(root) -> bool:
queue = deque([(root.left, root.right)]) # seed with the first pair
 
while queue:
left, right = queue.popleft()
 
if not left and not right: continue # both None → fine, skip
if not left or not right: return False # one None → fail
if left.val != right.val: return False # value mismatch
 
queue.append((left.left, right.right)) # outer pair
queue.append((left.right, right.left)) # inner pair
 
return True

The BFS version avoids recursion stack overflow on very deep trees. The logic is identical to the recursive version — you're just explicitly managing the "stack" of pairs to check using a deque instead of the call stack. Space complexity is O(w) where w is max tree width, vs O(h) for the recursive version.

05 · Real-world analogy

Think of a folded piece of paper

Imagine printing the tree on paper and folding it vertically down the middle through the root. A symmetric tree is one where every node on the left half lands exactly on top of a matching node on the right half — same value, same position relative to the fold. The isMirror function is simply checking, for each such pair of overlapping nodes, that their values agree and that their children will also overlap correctly when you unfold one level further.

Key insight: You never check the root against itself. The root is the fold line — you always compare nodes across the fold in pairs. That's why the entry point is isMirror(root.left, root.right), not isMirror(root, root).

06 · Understanding the code line by line

What every line is actually doing

The three-condition return statement is the entire algorithm. Each condition is a gate — all three must pass for the pair to be considered a valid mirror.

def isMirror(left, right) -> bool:
 
# GATE 1: structural — both absent = valid empty mirror
if not left and not right: return True
 
# GATE 2: structural — one absent, one present = structure mismatch
if not left or not right: return False
 
# Both nodes exist. Check 3 things — all must be True:
return (
left.val == right.val # GATE 3: value match at this level
and isMirror(left.left, right.right) # recurse: outer pair
and isMirror(left.right, right.left) # recurse: inner pair
)
Why does Gate 1 (both None) come before Gate 2 (one None)?
If you checked Gate 2 first with if not left or not right, it would also trigger when both are None (since not None is True). You'd return False for a valid symmetric leaf. Gate 1 must come first to catch the "both absent" case and return True before Gate 2 ever fires.
What does short-circuit evaluation mean here, and why does it matter?
Python's and stops evaluating as soon as it sees a False. So if left.val == right.val is False, the two recursive calls are never made — the whole expression immediately returns False. This means a mismatch near the top of the tree short-circuits all the work that would have been done in the subtrees.
Why not just serialize both subtrees and compare the strings?
Serialization works but has pitfalls: you must include null markers in the string, or trees with different structures but same traversal output can compare equal. The recursive approach is structurally guaranteed to be correct and more readable. Serialization is a valid interview approach if you handle nulls carefully (e.g., pre-order with sentinels).

08 · Full recursive trace — asymmetric tree

Walking through [1, 2, 2, null, 3, null, 3]

One call returns False — and due to short-circuit evaluation, everything above it immediately collapses to False too.

Entry

isSymmetric(root=1) → isMirror(node(2), node(2))

Root is valid. Call isMirror with both children.

Check

isMirror(L=node(2), R=node(2))

Neither None. 2 == 2 ✓. Now check outer pair first: isMirror(L.left=None, R.right=None).

Outer

isMirror(None, None) — L.left vs R.right

Both None → Gate 1 → returns True. Outer pair passes. Now evaluate the inner pair.

Fail

isMirror(L.right=node(3), R.left=None) — inner pair

Left is node(3), right is None — one null, one not. Gate 2 fires → returns False immediately. No deeper calls made.

Done

False propagates all the way up → isSymmetric returns False ✕

The and chain at the isMirror(2,2) level sees the inner pair returned False and short-circuits the entire call to False.

09 · Common mistakes

Errors to avoid in interviews

✕

Comparing same-side children instead of crossing the axis

Calling isMirror(left.left, right.left) is wrong. You must cross the axis: outer pair is left.left vs right.right; inner pair is left.right vs right.left. This single-line mistake is the most common interview error on this problem.

✕

Calling isMirror(root, root) — seeding with the root twice

This will always return True for any valid tree — you're comparing a node to its own copy. The correct seed is isMirror(root.left, root.right) because symmetry is about the two subtrees, not the root with itself.

✕

Putting Gate 2 before Gate 1 in the base cases

If you write if not left or not right: return False first, this also fires when both are None (since not None is True), incorrectly returning False for a perfectly valid empty-subtree pair. Gate 1 (both None → True) must always come first.

✕

Only checking values, not structure

A tree with mirrored values but different structure (one side has an extra child) is NOT symmetric. The base case if not left or not right: return False catches structural mismatches before value comparisons ever run.

10 · Complexity — the full explanation

Why O(n) time and O(h) space?

Time
O(n)
Each node participates in exactly one isMirror call as one half of one pair. n nodes → n/2 pairs → n/2 calls. n/2 ∈ O(n). Short-circuiting on asymmetric trees can exit well below n, but worst case (symmetric tree) is O(n).
Space
O(h)
The recursion stack depth equals the tree height h. O(log n) balanced, O(n) skewed. The iterative BFS version uses O(w) queue space where w is max width — could be O(n) for a wide tree but avoids stack depth issues.

Interview tip: Always clarify whether the space refers to the call stack (recursive) or the queue (iterative). For very deep trees the recursive approach can hit Python's default recursion limit (~1000). Mention the iterative BFS as a safe production alternative when tree depth is unknown.

11 · Interview Q&A

Questions you'll actually get asked

Can you solve this without recursion?
Yes — use an iterative BFS with a deque of pairs. Seed the queue with (root.left, root.right). Each iteration pops one pair, checks the same three conditions (both None → continue; one None → return False; values unequal → return False), then pushes the outer pair (left.left, right.right) and inner pair (left.right, right.left). Return True if the queue empties without failure.
What if the tree has only one node?
A single-node tree is symmetric by definition. isMirror(None, None) is called immediately — both are None, Gate 1 fires, returns True. No special case needed; the algorithm handles it automatically.
Does node order matter — what about equal values in the wrong positions?
Yes — values alone are not enough. The structure must also be mirrored. isMirror catches structural mismatches at Gate 2: if left exists but right is None, it returns False regardless of values. This means [1,2,2,null,3,null,3] is asymmetric even though all values appear mirrored.
How is this different from checking if two trees are identical?
Checking two identical trees compares left.left with right.left and left.right with right.right — same-side children. Symmetry crosses the axis: left.left is compared with right.right, and left.right with right.left. Exactly one line of the recursive call differs between the two problems.
